Hexo+GitHub搭建个人博客

准备工作

1.注册一个GitHub帐号
2.安装node.js
3.安装Git客户端

Hexo搭建博客

一、利用npm安装hexo

npm install hexo-cli -g

我们安装的node.js是自带了npm的,所以我们不需要再安装npm,如果出现’不是内部命令提示’请配置系统环境变量,如果出现安装失败或者下载到一般失败了(应该是被墙了),请下载淘宝镜像
关于npm和cnpm教程

二、初始化Hexo

hexo init blog

在你需要安装的位置(如:G:),打开我们下载的Git客户端(空白位置鼠标右击–>Git Bash Here)输入以上命令,Hexo就会先自动创建一个blog文件夹,然后将初始化文件下载到这个文件夹中

cd blog

切换到blog文件夹

npm install

接着输入以上命令 安装Hexo的依赖文件,安装完成后至此我们本地的Hexo博客就已经搭建好了. 我们来看看成果吧!

hexo server

启动hexo服务,
现在打开你的浏览器输入 它提示你的 localhost:4000
如果 hexo serve 提示找不到该指令,请输入下面指令:

npm install hexo -server --save

三、结合GitHub创建博客

1.创建页面仓库

2.生成SSH密匙(运用git工具)

ssh-keygen -t rsa -C "你的邮箱地址"

(>按3个回车，密码为空。)

3.在Github上添加SSH

生成密匙后,在 C:/Users/用户名/.ssh/下,会得到两个文件 id_rsa 和 id_rsa.pub,打开id_rsa.pub文件全部复制(CTRL +A ,CTRL + C) 进入Github的settings 粘贴进入随便写.



密匙的作用:就是用来向git推从信息的. 它是一种安全的传输模式 发布的时候要用到它

4.配置hexo

现在我们打开我们初始化的blog文件夹,这个文件加的目录结构是这样的

├── .deploy #需要部署的文件
├── node_modules #Hexo插件
├── public #生成的静态网页文件
├── scaffolds #模板
├── source #博客正文和其他源文件，404、favicon、CNAME 都应该放在这里
| ├── _drafts #草稿
| └── _posts #文章
├── themes #主题
├── _config.yml #全局配置文件
└── package.json

配置全局配置文件

博客配置
title:  #标题
subtitle:  #副标题
description:  #站点描述，给搜索引擎看的
author:  #作者
email:  #电子邮箱
language: zh-CN #语言
地址配置
url: # 博客地址  与github 一致
root: / 
permalink: :year/:month/:day/:title/ #文章的链接格式
tag_dir: tags #标签目录
archive_dir: archives #存档目录
category_dir: categories #分类目录
code_dir: downloads/code
permalink_defaults:
目录配置
source_dir: source #源文件目录
public_dir: public #生成的网页文件目录
写作配置
new_post_name: :title.md #新文章标题
default_layout: post #模板，包括 post、page、photo、draft（文章、页面、照片、草稿）
titlecase: false #标题转换成大写
external_link: true #在新选项卡中打开连接
filename_case: 0
render_drafts: false
post_asset_folder: false
relative_link: false
highlight: #语法高亮
    enable: true #是否启用
    line_number: true #显示行号
    tab_replace:
分类和标签
default_category: uncategorized #默认分类
category_map:
tag_map:
分页
2: 开启分页
1: 禁用分页
0: 全部禁用
archive: 2
category: 2
tag: 2
本地服务器
port: 4000 #端口号
server_ip: localhost #IP 地址
logger: false
logger_format: dev
日期时间格式
date_format: YYYY-MM-DD #参考http://momentjs.com/docs/#/displaying/format/
time_format: H:mm:ss
分页
per_page: 10 #每页文章数，设置成 0 禁用分页
pagination_dir: page
评论，替换为多说
disqus_shortname:
拓展插件
theme: landscape-plus #主题
exclude_generator:
plugins: #插件，例如生成 RSS 和站点地图的
    - hexo-generator-feed
    - hexo-generator-sitemap
部署
deploy:
    type: git
    repo: 刚刚github创库地址.git
    branch: master

根据说明配置主题、评论、打赏、RSS等功能

5.编辑文章

1.新建文章
当输入以下命令 会在hexo文件夹 _posts 目录下生成 “标题.md”的文件,可用Markdown语法书写.

hexo new "标题"

2.预览
当你编辑完文章后可直接打开git工具 生成网页,本地预览

hexo server

3.发布
文章写完后可用以下命令发布

hexo generate
hexo deploy

当出现以下信息说明发布成功

[info] Deploy done: git

这个时候你就可以访问你的”xxxxx.github.io”看效果喽!至此一个简单的hexo + github的博客就完成了!
